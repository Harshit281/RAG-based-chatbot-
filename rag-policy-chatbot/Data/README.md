# Data — Policy Knowledge Base

This directory contains the **source policy documents** and **auto-generated cache files** used by the RAG chatbot.

---

## Contents

### `policy_data.db` — OPP-115 Privacy Policy Database

The primary knowledge base file, stored as a SQLite database. It contains the **OPP-115** dataset fetched from Hugging Face, where each row represents a single privacy policy segment.

**Schema (table: `policy_data`):**

| Column | Type | Description | Example |
|---|---|---|---|
| `ID` | TEXT (PK) | Unique segment identifier | `OPP-0001` |
| `Topic` | TEXT | Privacy policy category (mapped from OPP-115 labels) | `First Party Collection and Use` |
| `Content` | TEXT | Full privacy policy text segment | `"Contact Us" Link If you contact us through the...` |

**Statistics:**

| Metric | Value |
|---|---|
| Total entries | 2,185 |
| Privacy categories | 12 |
| Source | [alzoubi36/opp_115](https://huggingface.co/datasets/alzoubi36/opp_115) (Hugging Face) |

**Topic distribution:**

| Topic | Count |
|---|---|
| First Party Collection and Use | 705 |
| User Choice and Control | 368 |
| Practice Not Covered | 231 |
| Other | 212 |
| International and Specific Audiences | 189 |
| Data Security | 128 |
| User Access Edit and Deletion | 113 |
| Third Party Sharing and Collection | 71 |
| Policy Change | 65 |
| Introductory/Generic | 61 |
| Data Retention | 29 |
| Do Not Track | 13 |

**Rebuilding the database:**

```bash
pip install datasets          # one-time install
python Scripts/prepare_opp115.py
```

This script drops and recreates the `policy_data` table, fetches the OPP-115 dataset from Hugging Face, and inserts all rows in a single atomic transaction.

---

### `cache/` — Auto-Generated Embeddings & Index Cache

This subdirectory is **created automatically** on the first run of the chatbot. It stores precomputed artifacts so that subsequent launches skip the expensive embedding and indexing steps.

| File | Format | Size (approx.) | Description |
|---|---|---|---|
| `embeddings.npy` | NumPy `.npy` binary | ~10–15 MB | Pre-computed dense vector embeddings for all 2,185 policy segments, generated by the `sentence-transformers` model. |
| `faiss.index` | FAISS binary (or Python pickle) | ~5 MB | Serialised FAISS `IndexFlatL2` vector index. If FAISS is unavailable, a pickled scikit-learn `NearestNeighbors` index is stored instead. |

> **Important:**
> - The `cache/` directory is listed in `.gitignore` and **should not be committed** to version control.
> - Delete this directory to **force a full rebuild** of embeddings and the vector index on the next launch.

## Extending the Knowledge Base

### Option A — Re-run the OPP-115 import

```bash
python Scripts/prepare_opp115.py
```

This drops and recreates the table from scratch using the latest OPP-115 data.

### Option B — Add rows manually

Use any SQLite client (e.g., [DB Browser for SQLite](https://sqlitebrowser.org/), the `sqlite3` CLI, or Python):

```sql
INSERT INTO policy_data (ID, Topic, Content)
VALUES ('CUSTOM-001', 'My Topic', 'Full policy text here.');
```

After adding or modifying data, delete `cache/` and restart the chatbot so embeddings and the index are rebuilt.

## Data Flow

```
  policy_data.db
       │
       │  loaded by Scripts/load_data.py (load_db)
       ▼
   PolicyData (list of dicts)
       │
       │  chunked by Scripts/chunk_data.py (uses Utils/text_cleaning.py)
       ▼
   List[Dict]  (chunks)
       │
       │  embedded by Scripts/embed_data.py
       ▼
   np.ndarray  ──► saved to cache/embeddings.npy
       │
       │  indexed by Scripts/vector_store.py
       ▼
   VectorStore  ──► saved to cache/faiss.index
```
