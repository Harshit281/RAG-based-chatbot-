# Data — Policy Knowledge Base

This directory contains the **source policy documents** and **auto-generated cache files** used by the RAG chatbot.

---

## Contents

### `policy_data.csv` — Source Knowledge Base

The primary knowledge base file. Each row represents a single company policy entry.

**Schema:**

| Column | Type | Description | Example |
|---|---|---|---|
| `ID` | Integer | Unique policy identifier | `1` |
| `Topic` | String | Short policy title / category | `Remote Work Policy` |
| `Content` | String | Full policy text (may include a `[Ref Code: XX-000]` tag for citation tracking) | `Employees can work remotely up to 3 days per week...` |

**Statistics:**

| Metric | Value |
|---|---|
| Total entries | 206 |
| File size | ~27 KB |
| Topics covered | Remote work, health insurance, IT security, expense reimbursement, leave policies, and more |

**Sample rows:**

| ID | Topic | Content (truncated) |
|---|---|---|
| 1 | Remote Work Policy | Employees can work remotely up to 3 days per week. Core collaboration hours are 10:00 AM to 3:00 PM EST. |
| 2 | Health Insurance | Standard medical coverage includes 80% of dental work and 100% of annual wellness checkups... |
| 4 | Expense Reimbursement | Travel expenses must be submitted via the Concur portal within 30 days of the trip completion... |

---

### `cache/` — Auto-Generated Embeddings & Index Cache

This subdirectory is **created automatically** on the first run of the chatbot. It stores precomputed artifacts so that subsequent launches skip the expensive embedding and indexing steps.

| File | Format | Size (approx.) | Description |
|---|---|---|---|
| `embeddings.npy` | NumPy `.npy` binary | ~1–2 MB | Pre-computed dense vector embeddings for all 206 policy chunks, generated by the `sentence-transformers` model. |
| `faiss.index` | FAISS binary (or Python pickle) | ~1 MB | Serialised FAISS `IndexFlatL2` vector index. If FAISS is unavailable, a pickled scikit-learn `NearestNeighbors` index is stored instead. |

> **Important:**
> - The `cache/` directory is listed in `.gitignore` and **should not be committed** to version control.
> - Delete this directory to **force a full rebuild** of embeddings and the vector index on the next launch.

## Adding New Policy Documents

To expand the knowledge base with new policy entries:

1. **Open** `policy_data.csv` in any spreadsheet editor or text editor.
2. **Append new rows** at the bottom, following the `ID,Topic,Content` format:
   ```csv
   207,New Policy Topic,"Full policy text here. [Ref Code: NP-001]"
   ```
3. **Delete the cache**: remove the entire `cache/` directory (or just the files inside it).
4. **Restart the chatbot**: run `python main.py` — embeddings and the vector index will be regenerated automatically.

> **Tip:** The `[Ref Code: XX-000]` tag inside the Content field is optional. If present, it will be extracted and shown as a source citation alongside the generated answer.

## Data Flow

```
  policy_data.csv
       │
       │  loaded by Scripts/load_data.py
       ▼
   pd.DataFrame
       │
       │  chunked by Scripts/chunk_data.py (uses Utils/text_cleaning.py)
       ▼
   List[Dict]  (chunks)
       │
       │  embedded by Scripts/embed_data.py
       ▼
   np.ndarray  ──► saved to cache/embeddings.npy
       │
       │  indexed by Scripts/vector_store.py
       ▼
   VectorStore  ──► saved to cache/faiss.index
```
